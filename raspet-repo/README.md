# 🐣 RasPet — 라즈베리파이 기반 캐릭터 육성형 게임 콘솔

> *다마고치 · 프린세스 메이커 콘셉트를 결합한 통합 임베디드 게임기*

[![CI](https://github.com/Gusle01/raspberrry-pi5/actions/workflows/ci.yml/badge.svg)](https://github.com/Gusle01/raspberrry-pi5/actions/workflows/ci.yml)

라즈베리파이 5 위에서 **카메라 · 초음파 센서 · 조이스틱 · OLED**를 하나의 애플리케이션으로 통합하여,
사용자가 자신만의 캐릭터를 키우는 휴대형 육성 게임기를 직접 제작하는 프로젝트입니다.
컴퓨터 비전(손 인식)과 게임 인공지능(오목)을 하나의 작품 안에서 다룹니다.

---

## 📌 프로젝트 개요

게임기를 켜면 OLED 화면에 자신의 캐릭터가 등장합니다. 캐릭터는 시간이 지남에 따라 배가 고파지고
스트레스를 받으며, 사용자는 미니게임을 플레이해 재화를 얻고 그 재화로 캐릭터를 돌보거나 꾸밉니다.
다양한 활동을 통해 체력·지력·매력 등의 능력치가 성장하고, **최종 능력치 조합에 따라 서로 다른 엔딩**에 도달합니다.

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 라즈베리파이 기반 캐릭터 육성형 게임 콘솔 (가칭: RasPet) |
| 분류 | 임베디드 시스템 / 컴퓨터 비전 / 게임 SW |
| 플랫폼 | Raspberry Pi 5 (RAM 4GB), Raspberry Pi OS 64-bit |
| 언어 | Python 3 |
| 과목 | 임베디드시스템 (담당교수: 윤수경) |
| 제작 | 202312689 박영주 |

## ✨ 주요 기능

### 🎮 캐릭터 육성 시스템
- **능력치(스탯):** 체력 · 근력 · 지력 · 매력 · 감수성 · 스트레스 · 행복도
- **시간 경과 요소(다마고치):** 포만도·청결도가 실제 시간에 따라 감소 → 방치하면 부정 이벤트 발생
- **육성 루프:** 활동 선택 → 능력치·상태 변화 → 캐릭터 돌보기 → 반복

### 🕹️ 미니게임
| 게임 | 입력 장치 | 핵심 기술 | 비고 |
|---|---|---|---|
| 가위바위보 | 카메라 | MediaPipe 손 인식 | **필수** |
| 오목 | 조이스틱 | Minimax + α-β 가지치기 AI | **필수** |
| 초음파 점프 | 초음파 센서 | 거리 → 점프 높이 제어 | 제안 |
| 스네이크 / 아이템 받기 | 조이스틱 | 클래식 아케이드 | 제안 |
| 색깔 찾기 | 카메라 | OpenCV 색상 인식 | 제안 |

### 🛒 상점 및 재화 시스템
미니게임 승리 → 재화 획득 → 식사·놀기 소비 또는 상점 구매 → 캐릭터 성장·꾸미기
(코스튬 · 소비 아이템 · 능력치 초기화권 · 성장 부스터 등)

### 🌟 추가 기능 (프린세스 메이커 응용) — 구현 완료
- **다중 엔딩** — 최종 능력치 조합 → 학자/예술가/모험가/스타/평범한 행복
- **랜덤 이벤트** — 미니게임 후·'탐험'에서 확률 발생 (행운·감기·영감 등)
- **도전 과제(업적)** — 조건 달성 시 해금·팝업·목록
- **성장 단계(진화)** — 능력치 누적에 따라 외형 변화 (알→아기→어린이→어른)
- **시간대 반영** — 아침/낮/저녁/밤 배경 색조·인사말
- **감정 표현** — 상태(배고픔·아픔·행복 등)에 따라 표정 변화

## 🏗️ 시스템 구성

전체 시스템은 **하드웨어 계층 → 소프트웨어 계층 → 애플리케이션 계층**의 3계층으로 구성됩니다.

```
┌─────────────────────────────────────────────────────────┐
│  애플리케이션 계층                                          │
│  캐릭터 육성 · 미니게임 · 상점/재화 · 엔딩                    │
├─────────────────────────────────────────────────────────┤
│  소프트웨어 계층 (Python 3)                                 │
│  Pygame 게임 루프 · OpenCV/MediaPipe 비전 · 오목 AI ·       │
│  luma.oled 출력 · gpiozero 제어 · SQLite/JSON 저장          │
├─────────────────────────────────────────────────────────┤
│  하드웨어 계층 (Raspberry Pi 5)                             │
│  CSI 카메라 · HC-SR04 · 조이스틱(MCP3008 ADC) · OLED · 부저 │
└─────────────────────────────────────────────────────────┘
```

## 🚀 설치 및 실행

> Raspberry Pi 5 / Raspberry Pi OS (64-bit) 기준입니다.

```bash
# 1) 저장소 클론
git clone https://github.com/Gusle01/raspberrry-pi5.git
cd raspberrry-pi5

# 2) 가상환경 (picamera2 등 시스템 패키지 사용을 위해 --system-site-packages 권장)
python3 -m venv .venv --system-site-packages
source .venv/bin/activate

# 3) 의존성 설치
pip install -r requirements.txt

# 4) 실행
python main.py
```

> 💡 `picamera2`는 Raspberry Pi OS에 기본 포함되어 있어 보통 `sudo apt install -y python3-picamera2`로 설치합니다.
> `MediaPipe`는 aarch64 환경에서 설치 제약이 있을 수 있으며, 이 경우 OpenCV 윤곽선 기반 대체 방식을 사용합니다. (자세한 내용은 `docs/project-plan.md` 5.1절)

### 💻 PC에서 개발 · 테스트하기

모든 하드웨어 접근은 `hardware/`·`vision/` 계층 뒤로 추상화되어 있고, 장치가 없으면
자동으로 더미로 폴백합니다. 덕분에 라즈베리파이가 없는 PC에서도 그대로 실행·테스트할 수 있습니다.

```bash
# 키보드로 조작하는 미리보기 창으로 실행 (방향키 이동 · Enter 확인 · Esc 취소)
RASPET_DUMMY=1 python main.py

# 장치/창 없이(헤드리스) 실행 — 자동화·CI용
RASPET_DUMMY=1 RASPET_HEADLESS=1 python main.py

# 단위 테스트 (하드웨어 불필요)
pytest
```

| 환경변수 | 효과 |
|---|---|
| `RASPET_DUMMY=1` | 실제 센서/카메라/OLED를 잡지 않고 더미로 동작 |
| `RASPET_HEADLESS=1` | 창·OLED 없이 게임 루프 구동 (테스트용) |

조작: **방향키/WASD** 이동, **Enter/Space/Z** 확인, **Esc/Backspace/X** 취소.
실기에서는 조이스틱 방향·버튼이 동일한 행동(action)으로 매핑됩니다.
