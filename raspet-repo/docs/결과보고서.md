> 라즈베리파이 5 기반 캐릭터 육성형 게임 콘솔 — 임베디드시스템 프로젝트

|---|---|

| 플랫폼 | Raspberry Pi 5 (4GB) · Raspberry Pi OS 64-bit |

## 1. 개요

다마고치(시간 경과 육성)와 프린세스 메이커(능력치 → 다중 엔딩) 콘셉트를 결합해, 128×64 OLED
화면 위에서 나만의 펫을 키우는 **휴대형 임베디드 게임기**를 직접 제작했다. 카메라(손 인식),
센서(거리·조도·온도), 아날로그 조이스틱, 키패드, LED, 부저, 7세그먼트를 **하나의 Pygame
애플리케이션**으로 통합했고, 모든 하드웨어는 추상화 계층 뒤에서 장치가 없으면 더미로 폴백하여
PC에서도 그대로 개발·테스트된다. (단위/통합 테스트 97케이스 + GitHub Actions CI)

## 2. 구현한 기능

### 캐릭터 육성 (코어)
- 능력치(체력·근력·지력·매력·감수성·스트레스·행복도)와 시간 경과(포만·청결 감소) 다마고치 루프
- 성장 단계(알→아기→어린이→어른) 진화 외형 변화
- 감정 표현: 상태(배고픔·아픔·행복·졸림 등)에 따른 표정, 더위=땀·추위=눈송이
- XP·레벨(전체 진행도) HUD, 다중 엔딩 + 방치 엔딩 + 세대 교체(업적·기록 계승)
- 랜덤 이벤트, 도전 과제(업적), 시간대(아침/낮/저녁/밤) 반영, 상점·재화

### 미니게임
- **가위바위보** — 카메라 손 인식(OpenCV 백엔드, 손가락 수·각도 필터)
- **오목** — Minimax + α-β 가지치기 AI
- **초음파 점프** — 거리로 플래피버드식 장애물 통과
- **색깔 찾기** — OpenCV 색상 인식 + LED 색 힌트
- **두더지 잡기** — 4×4 키패드 OLED 격자(미연결 시 3버튼 자동 폴백)
- 미니게임별 **베스트 기록** 저장·표시

### 하드웨어 연동 (이번 작업의 핵심)
- BMP180 온도 → 더위/추위 반응, LDR 조도 → 수면/기상(낮·밤 판정)
- LED 4개: 메인=온도색 / 색깔찾기=색 힌트 / 두더지=표시등
- 4자리 7세그먼트(옵션): 평소=온도, 시간제한 게임=남은 시간
- 조이스틱 SW를 GPIO 디지털+내부풀업으로 확정(ADC floating 문제 해결)
- 카메라 미리보기 창, RGB/BGR 색 보정 등 실측 기반 안정화

## 3. 기능 ↔ 사용 부품 표

| 구현 기능 | 사용 부품 | 인터페이스 / 핀 (BCM) |
|---|---|---|
| 게임 화면 출력(캐릭터·메뉴·게임) | OLED SSD1306 128×64 | I2C, SDA=GPIO2 · SCL=GPIO3, 주소 0x3C |
| 메뉴·게임 조작 | 2축 아날로그 조이스틱 | VRx→MCP3008 CH0, VRy→CH1 / SW=GPIO25(디지털·내부풀업) |
| 아날로그 값 읽기(조이스틱·조도) | MCP3008 ADC | SPI0 (CLK11·MISO9·MOSI10·CE0=8) |
| 가위바위보·색깔찾기·미리보기 | CSI 카메라(OV5647) | CSI 커넥터 + OpenCV/picamera2 |
| 초음파 점프 | 초음파 HC-SR04 | TRIG=GPIO23, ECHO=GPIO24(분압 필수) |
| 효과음 | 피에조 부저 | GPIO18 (PWM) |
| 수면/기상(낮·밤) 반응 | 조도센서 LDR + 10kΩ | MCP3008 CH2 (분압) |
| 더위/추위 표정 | BMP180 (온도·기압) | I2C 공유(GPIO2·3), 주소 0x77 |
| 두더지 잡기 격자·메뉴 입력 | 4×4 매트릭스 키패드 | 행 GPIO26·17·27·22 / 열 GPIO12·16·20·21 |
| 두더지(키패드 미사용 시) | 푸시버튼 ×3 | GPIO16·20·21 (내부 풀업) |
| 온도색·색 힌트·두더지 표시등 | LED ×4 (초록·빨강·노랑·파랑) | GPIO5·6·13·19 (각 220~330Ω) |
| 온도·남은시간 표시(옵션) | 4자리 7세그 HSN-3643AS + 74HC595 | DS14·SHCP15·STCP19, 자릿수 GPIO5·6·13·4(트랜지스터) |
| OS·세이브 저장 | microSD 32GB↑ | SQLite/JSON |

> ⚠️ LED 4핀(5·6·13·19)과 7세그는 **같은 핀을 공유**하므로 동시 사용 불가 —
> 기본은 LED 모드, `RASPET_7SEG=1`일 때만 7세그로 전환된다.

## 4. 기술적 포인트 / 트러블슈팅

| 문제 | 해결 |
|---|---|
| Pi 5에서 `RPi.GPIO` 미동작 | `gpiozero` + `lgpio` 백엔드로 전환 |
| MediaPipe가 Python 3.13/aarch64에서 설치 불가 | OpenCV 윤곽선·각도 기반 손 인식으로 대체 |
| picamera2 RGB888이 실제 BGR로 들어와 색 반전 | R↔B 채널 보정 |
| 조이스틱 SW를 ADC로 읽으니 풀업 없이 값이 floating | SW를 GPIO25 디지털+내부풀업으로 이전 |
| 핀 부족 | LED/7세그 핀 공유, 키패드/버튼 핀 공유로 모드 전환 설계 |
| 장치 없는 PC 개발 | 모든 하드웨어 더미 폴백 + 97케이스 테스트·CI |

## 5. 기능별 구현 방식 (요약)

> 모듈 단위 상세 설명은 [`코드북.md`](코드북.md) 참고. 여기서는 기능별 핵심 구현만 정리한다.

| 기능 | 구현 방식 (핵심 기술) |
|---|---|
| 캐릭터 육성(다마고치) | 마지막 플레이 시각과 현재의 차이로 포만·청결을 비례 감소(`needs.apply_time_decay`), 돌보기 1회=하루(`advance_day`) |
| 감정 표현 | `config.MOODS` 규칙을 위에서부터 검사하는 우선순위 룰 엔진(`mood.compute_mood`), 표정은 외부 이미지 없이 절차적 도형으로 렌더 |
| 진화 / XP·레벨 | 핵심 스탯 합 임계로 성장 단계 전환(`update_stage`), XP는 선형 곡선의 독립 레이어(`progression`) |
| 다중·방치 엔딩 | 능력치 조합으로 엔딩 분기(`determine_ending`), 연속 방치 일수·스트레스로 강제 엔딩(`check_forced_ending`) |
| 가위바위보 | 카메라 → 피부색 마스킹·윤곽선·볼록결함으로 손가락 수 추정(OpenCV), 여러 프레임 다수결로 안정화 |
| 오목 | Minimax + α-β 가지치기, 길이 5 윈도우 평가 함수, 후보 축소·정렬로 Pi 5에서 수 초 내 응답 |
| 초음파 점프 | HC-SR04 거리 → 0~1 높이 매핑, 0~1 정규화 좌표의 순수 파이썬 충돌 엔진 |
| 색깔 찾기 | HSV 변환 후 목표 색 범위 픽셀 비율 판정(OpenCV), 흑백 OLED 보완용 LED 색 힌트 |
| 두더지 잡기 | 시간 보간으로 난이도 상승하는 순수 파이썬 엔진, 키패드(16칸 격자)/버튼(3구멍) 자동 전환 |
| 환경 반응 | LDR(MCP3008 CH2) 조도→수면/낮밤, BMP180(smbus 직접 드라이버)→더위·추위 표정 |
| 입출력 통합 | `GameContext` 단일 게이트웨이가 키보드/조이스틱/버튼/키패드를 엣지 트리거 행동으로 정규화 |
| 이식성·테스트 | 모든 하드웨어 `create_*()` 더미 폴백 → PC·CI에서 동일 코드 실행, 97케이스 + GitHub Actions |

## 6. 결론

목표했던 **카메라 비전 + 다중 센서 + 게임 AI를 하나의 임베디드 게임기로 통합**을 달성했다.
필수 기능(가위바위보·오목)에 더해 초음파 점프·색깔 찾기·두더지 잡기와 BMP180/LDR 환경 반응,
LED·7세그 출력, 다중/방치 엔딩까지 구현 완료했다. 하드웨어 추상화와 자동 폴백 설계로 부품
구성이 달라져도 코드 수정 없이 `config.py`만으로 대응할 수 있다.
