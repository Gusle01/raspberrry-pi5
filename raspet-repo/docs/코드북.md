## 0. 읽는 법 / 전체 아키텍처

### 0.1 3계층 구조

```
┌─────────────────────────────────────────────────────────┐
│  애플리케이션 계층  (app.py · core/scenes.py)              │
│  캐릭터 육성 · 미니게임 런처 · 상점/재화 · 엔딩            │
├─────────────────────────────────────────────────────────┤
│  소프트웨어 계층  (core/ · character/ · ai/ · minigames/) │
│  Pygame 게임 루프 · 룰 엔진 · 오목 AI · 저장              │
├─────────────────────────────────────────────────────────┤
│  하드웨어 추상화 계층  (hardware/ · vision/)              │
│  OLED · 조이스틱 · 센서 · 키패드 · LED · 7세그 · 카메라   │
└─────────────────────────────────────────────────────────┘
```

### 0.2 코드 전체를 관통하는 두 가지 설계 원칙

1. **GameContext 단일 게이트웨이** (`core/context.py`)
   모든 씬·미니게임은 pygame이나 GPIO를 **직접 만지지 않고** `ctx`(GameContext)를 통해서만 입출력한다.
   덕분에 게임 로직과 하드웨어가 완전히 분리되어, 하드웨어를 바꿔도 로직 코드는 그대로다.

2. **더미 폴백(dummy fallback) 패턴** (`hardware/`, `vision/` 전체)
   모든 하드웨어 모듈은 `create_*()` 팩토리를 두고, 라이브러리가 없거나·장치 생성이 실패하면
   같은 인터페이스의 **더미(no-op) 객체**를 돌려준다. 그래서 라즈베리파이가 없는 PC·CI에서도
   동일한 코드가 깨지지 않고 실행·테스트된다. (단위/통합 테스트 97케이스 + GitHub Actions)

### 0.3 디렉터리 맵

| 디렉터리 | 역할 | 본 문서 절 |
|---|---|---|
| `core/` | 게임 루프·씬 머신·컨텍스트·스프라이트·시간대·효과음 | §1 |
| `character/` | 능력치·돌봄(시간경과)·무드·진화·XP·엔딩 + `events`/`achievements` | §2 |
| `minigames/` · `ai/` | 6종 미니게임 + 오목 AI | §3 |
| `hardware/` | OLED·조이스틱·초음파·버튼·부저·LED·키패드·7세그·환경센서 | §4 |
| `vision/` | 카메라 캡처·손 인식·미리보기 | §5 |
| `storage/` · `shop/` | 세이브(JSON/SQLite)·상점 | §6 |

## 1. 게임 코어 (core/) 및 부트스트랩

### `app.py` — 애플리케이션 부트스트랩
- **역할**: 하드웨어를 초기화하고 세이브를 불러온 뒤 게임 루프를 시작하는 진입점.
- **핵심 기능 및 구현**:
  - `build_hardware()`(`app.py:26`)가 디스플레이·조이스틱·초음파·부저·LED·버튼·키패드·환경센서·카메라·손인식·7세그 등 **11종 장치를 dict로** 묶어 반환. 각 `create_*()`가 더미 폴백을 보장한다.
  - **핀 공유 충돌을 코드로 회피**: `KEYPAD_ENABLED`이면 버튼을 더미로(핀 16·20·21 공유), `SEG_ENABLED`이면 LED를 더미로(핀 5·6·13을 7세그 자릿수로 재사용) 둔다(`app.py:29-31`).
  - `main()`(`app.py:47`)이 하드웨어 → `GameContext` → `save.load_character()`(마지막 저장 시각 복원) → `GameLoop` → `run()`을 구성하고 `finally`에서 자원을 정리한다.

### `core/game_loop.py` — 메인 게임 루프
- **역할**: 입력→갱신→렌더→출력→FPS 제한을 반복하는 Pygame 메인 루프이자 앱 상태 컨테이너.
- **핵심 기능 및 구현**:
  - `GameLoop`(`game_loop.py:19`)가 캐릭터·`Shop`·`SceneManager`·난수원을 보유하고 `ctx.app = self`로 씬이 앱 상태를 역참조하게 한다.
  - **다마고치 핵심**: 생성 시 `last_saved_ts`가 있으면 `needs.apply_time_decay()`로 **마지막 플레이 이후 경과 시간만큼** 포만·청결을 감소시킨다(`game_loop.py:29-30`).
  - `run()`의 한 프레임: `poll`(입력)→`tick`(dt)→씬의 `handle_input/update/render`→`present`(창+OLED)→`update_temp_led`(온도→LED)→`seg_show_temp`(7세그). 종료 시 자동 저장.

### `core/context.py` — 게임 컨텍스트 (입출력·하드웨어 게이트웨이) ★핵심
- **역할**: 입력·렌더·시간·하드웨어를 한곳에서 추상화하는 핵심 계층.
- **핵심 기능 및 구현**:
  - **고정 가상 캔버스 + 출력 분리**: 모든 씬은 OLED 해상도(`GAME_W×GAME_H`) 서피스에만 그린다. `present()`(`context.py:302`)가 이를 PC 창에는 비율 유지 **정수배 확대(레터박스)** 로, OLED에는 PIL 변환해 1:1 전송 → 한 번 그려 두 출력을 지원.
  - **엣지 트리거 입력 통합**: `poll()`(`context.py:154`)이 키보드·조이스틱(전환 순간만)·물리버튼·키패드를 단일 행동 문자열(`up/down/left/right/a/b`)로 정규화. `script` 주입 시 자동 테스트 모드.
  - **상황별 매핑 전환**: `set_button_actions()`/`set_keypad_mode()`로 버튼·키패드의 역할을 메뉴↔게임용으로 전환(`context.py:227-238`).
  - **한글 폰트 해석**: 번들 픽셀폰트(Galmuri) > 시스템 한글폰트 우선순위 탐색(`context.py:32-110`).
  - **하드웨어 헬퍼 위임**: `beep`·`set_color_led`·`temp_led_color`·`seg_show_*`·`distance_cm`·`classify_hand`·환경센서 폴링 캐시(`ENV_POLL_MS` 주기) 등을 제공.

### `core/scene.py` / `core/scenes.py` — 씬(상태) 머신
- **역할**: 모든 화면의 추상 인터페이스(`scene.py`)와 실제 화면·미니게임 런처(`scenes.py`, 677줄).
- **핵심 기능 및 구현**:
  - `Scene(ABC)`가 `handle_input`·`update`·`render` 3단 계약을 정의, `SceneManager`가 현재 씬을 교체한다.
  - `TitleScene`→`MenuScene`(돌보기/미니게임/상점/탐험/업적/엔딩) 중심. `MenuScene`은 펫·이름·재화·키운 일수(D+)·온도·레벨바를 한 화면에 배치.
  - `CareScene`(`scenes.py:318`): 먹이/씻기/놀이/훈련. **행동 1회 = 하루** → `needs.advance_day()` 후 `check_forced_ending()`(방치/스트레스 강제 엔딩) 검사 + XP 적립.
  - **미니게임 런처** `play_and_reward()`(`scenes.py:567`): 6종을 이름으로 디스패치 실행하고 보상→재화·XP·베스트기록·랜덤이벤트·업적까지 일괄 처리. 미니게임끼리 서로 의존하지 않도록 **이 앱 계층에서만 조립**.
  - `EndingScene`은 확인 시 `reset_life()`로 **새 세대 초기화**(업적·베스트기록·이름은 계승).

### `core/sprite.py` — 캐릭터 스프라이트 (절차적 도형)
- **역할**: 외부 이미지 없이 **도형만으로** 펫을 그리며 성장 단계·무드에 따라 외형/표정을 바꾼다.
- **핵심 기능 및 구현**:
  - `draw_pet()`(`sprite.py:16`)가 성장 단계(0~3: 알→아기→어린이→어른)별로 외형 분기(반지름 `(7+stage*3)`, 2단계 귀·3단계 발).
  - `_draw_face`(`sprite.py:66`)가 `_FACES` 딕셔너리에서 무드 id→표정 함수를 디스패치. **13종 표정**(happy/sad/sick/hungry/sleepy/asleep/hot/cold…)을 눈·입 도형 변형으로 구현 — 예: `_face_hot`=혀+땀, `_face_cold`=눈송이.

### `core/daytime.py` — 시간대 판정
- **핵심 기능 및 구현**: `current_period()`가 **조도센서 우선** 로직 구현 — 센서가 어두우면 'night'(수면), 밝으면 시계가 밤이어도 'day'(기상). 센서 없으면 시스템 시계로 폴백. 시간대별 배경 색조·인사말 반환.

### `core/sfx.py` — 효과음
- **핵심 기능 및 구현**: `(주파수,길이)` 음 목록을 `ctx.beep`으로 순차 재생. move/confirm/cancel/win/event/achievement 6종을 음높이·음수로 차별화(승리·업적=상승 3음 팡파레). 부저 부재 시 자동 무음.

## 2. 캐릭터 육성 시스템 (character/)

### `character.py` — 데이터 모델 + 능력치
- **핵심 기능 및 구현**:
  - `@dataclass Character`(`character.py:18`)가 모든 상태를 한 객체로 관리: 능력치(체력·근력·지력·매력·감수성·스트레스·행복도)·돌봄 상태(포만·청결)·진행(재화·인벤·`stage`·`xp`·`day`)·통계(`best_scores`·`achievements`). `_clamp()`로 0~`STAT_MAX` 범위 강제.
  - `grow()`(`character.py:62`): 스트레스가 임계 이상이면 성장 효율을 낮춤. `train()`은 성장+스트레스 증가 트레이드오프.
  - **진화** `update_stage()`(`character.py:98`): 핵심 4스탯 합을 `STAGE_THRESHOLDS`와 비교해 단계 갱신(능력치 변동마다 자동).
  - **세대 교체** `reset_life()`(`character.py:128`): 능력치·일수·재화는 초기화하되 **업적·베스트기록·이름은 평생 계승**.

### `needs.py` — 다마고치 시간경과·돌봄
- **핵심 기능 및 구현**:
  - `apply_time_decay()`(`needs.py:9`): 경과 시간 × `NEED_DECAY_PER_HOUR`로 포만·청결을 비례 감소. 장기 방치 시 행복·체력 추가 감소.
  - `advance_day()`(`needs.py:41`): "돌보기 1회=하루" 규칙. `day`+1, 매일 포만·청결 감소, **포만/청결/행복 연속 0 일수**(`zero_days`)를 누적 → 방치 엔딩 입력.
  - 돌봄 3종 `feed`/`clean`/`play_with`를 `apply_effects()`로 적용.

### `mood.py` — 무드(표정) 룰 엔진
- **핵심 기능 및 구현**:
  - `compute_mood()`(`mood.py:44`): `config.MOODS` 규칙을 위에서부터 검사해 **모든 조건(AND)을 만족하는 첫 규칙**의 무드를 채택(우선순위 룰 엔진). 마지막 `neutral`이 기본값.
  - 조건 `(신호명, 연산자, 값)`을 `_OPS`로 평가. 신호는 캐릭터 속성·시간대·환경(온도/수면/조도)에서 수집. **규칙을 코드가 아닌 config 데이터로** 분리.
  - 센서 미연결(None) 시 해당 환경 규칙은 거짓 처리 → 안전.

### `progression.py` — XP·레벨 (전체 진행도)
- **핵심 기능 및 구현**: 능력치·진화와 **독립된 레이어**. `xp_for_level()`은 선형 증가 곡선(`BASE+(L-1)*STEP`), `level_for_xp()`로 역산, `title_for_level()`로 타이틀, `progress()`로 HUD 진행 바 비율 반환.

### `endings.py` — 다중/강제 엔딩
- **핵심 기능 및 구현**:
  - `determine_ending()`(`endings.py:9`): `config.ENDINGS`를 순회해 `requires`(능력치≥값)를 모두 만족하는 첫 엔딩(학자/예술가/모험가/스타/평범한 행복) 반환.
  - `check_forced_ending()`(`endings.py:24`): 스트레스 최대=즉시 가출, 포만/청결/행복이 `NEGLECT_ENDING_DAYS`일 이상 연속 0이면 방치 엔딩(도둑/질병/방랑자).

### `events.py` / `achievements.py`
- **랜덤 이벤트**: `pick_event()`가 `weight` 기반 `rng.choices`로 추첨, `apply_event()`로 효과 적용. `maybe_trigger()`는 `EVENT_CHANCE` 확률(미니게임 후), `force_event()`는 확정 발생(탐험).
- **업적**: `check_and_unlock()`이 조건(`stat_total` 포함) 만족 업적을 해금·반환. 정의는 `config.ACHIEVEMENTS`, 해금 목록은 세대 교체에도 영속.

## 3. 미니게임 (minigames/ · ai/)

모든 미니게임은 `MiniGame(ABC)`(`base.py`)를 상속하며 `play() -> int`(획득 재화 반환) 계약을 따른다.
대부분 **순수 파이썬 엔진**(게임 규칙)과 **`ctx` 어댑터**(입출력)를 분리해 하드웨어 없이 테스트된다.

### 3.1 가위바위보 (`rps.py` + `vision/hand.py`) — 필수, 카메라 비전
- **기능**: 카메라로 손을 인식해 가위/바위/보를 판정하고 컴퓨터의 무작위 선택과 대결.
- **구현**:
  - **인식 백엔드 2단계 + 폴백**: ① MediaPipe Hands(21 랜드마크, 손목까지 거리로 회전 불변 손가락 카운트, `hand.py:78-108`) → ② OpenCV(설치 불가 환경의 실제 경로): `YCrCb` 피부색 마스킹 → 모폴로지 → **윤곽선 solidity**(꽉 차면 바위) → **볼록 결함(convexity defect)** 의 깊이(>20px)+사잇각(≤90°) 필터로 손가락 수 추정(`hand.py:138-202`) → ③ 비전 불가 시 버튼(`←바위 ↑가위 →보`).
  - 손가락 수 → 제스처(0~1=바위, 2~3=가위, 4~5=보).
  - **안정화**: 실시간 미리보기 + 5초 카운트다운 후 캡처, 여러 프레임을 **다수결**(`majority_gesture`)로 확정(`rps.py:_camera_choice`).
  - 임계값은 진단 도구 `tools/rps_probe.py`로 실측 보정.

### 3.2 오목 (`omok.py` + `ai/omok_ai.py`) — 필수, 게임 AI
- **기능**: 사람(선공) vs AI가 번갈아 두어 먼저 5목을 만들면 승리.
- **구현**:
  - 진행부 `omok.py`는 입력기 `move_provider`·렌더러 `renderer`를 **주입**받아 하드웨어와 분리. 보드는 2차원 리스트(0/1/2), `place`가 무효 좌표에 `ValueError`.
  - **AI**(`omok_ai.py`)는 **Minimax + 알파-베타 가지치기**:
    - 난이도=탐색 깊이/후보 수(easy 깊이1 ~ hard 깊이3), Pi 5에서 수 초 내 완료하도록 후보 제한.
    - `best_move`: 즉시 이기는 수 → 상대 즉승 방어 → 알파-베타 탐색 순(`omok_ai.py:45-82`).
    - **평가 함수**: 길이 5 윈도우를 4방향으로 훑어 돌 개수별 점수(1·10·100·1000·10만), 상대 위협은 1.1배 가중(방어적).
    - **후보 축소**: 기존 돌 주변(체비쇼프 거리 ≤2) 빈칸만, "공격+방어 가치"로 정렬해 상위 N개만 탐색.

### 3.3 초음파 점프 (`jump.py`) — 초음파 센서
- **기능**: HC-SR04로 손 높이를 읽어 캐릭터를 위아래로 움직여 파이프 구멍을 통과(플래피버드식).
- **구현**:
  - `height_from_distance()`가 거리(cm)를 0.0~1.0 높이로 선형 매핑(가까울수록 높음).
  - 충돌 로직 `JumpEngine`은 0~1 정규화 좌표의 순수 파이썬(하드웨어 없이 테스트). `_spawn`이 **항상 통과 가능하고 양쪽 파이프가 남는** 구멍을 무작위 생성. `step`에서 플레이어 x를 지나는 순간 몸 전체가 구멍 안인지로 충돌 판정.
  - 통과 개수 × 보상.

### 3.4 색깔 찾기 (`color_hunt.py`) — 카메라 + LED 힌트
- **기능**: 제시된 색을 띤 사물을 제한 시간 안에 카메라에 비추면 성공.
- **구현**:
  - `color_match()`가 프레임을 **HSV로 변환** 후 목표 색 범위(`cv2.inRange`) 픽셀 비율이 임계(5%) 이상이면 인식.
  - **흑백 OLED 한계 보완**: 찾을 색을 **LED로 점등**해 힌트(`set_color_led` + `config.COLOR_HUNT_LED`). 7세그에 남은 시간 표시.
  - `detector` 주입으로 OpenCV 없이도 테스트.

### 3.5 두더지 잡기 (`whack_a_mole.py`) — 키패드/버튼
- **기능**: 두더지가 나온 칸을 제한 시간 안에 눌러 잡는 30초 반응 게임.
- **구현**:
  - **입력 모드 자동 전환**: 4×4 키패드 연결 시 OLED **16칸 격자**(키 라벨 표시), 없으면 **버튼 3개**의 3구멍 모드로 자동 폴백(`play()`의 `grid_mode` 분기).
  - 게임 로직 `WhackEngine`은 순수 파이썬: 시간 경과에 따라 등장·수명을 **보간(`_lerp`)** 해 난이도 상승, **콤보 보너스**, **함정**(누르면 감점), 후반 동시 2마리.
  - **LED·부저 피드백**: 등장/명중/함정/놓침을 각기 다른 음으로, LED로 콤보·함정·두더지 표시. 베스트 갱신 시 "신기록!".

### 3.6 스네이크 (`snake.py`)
- **구현**: 순수 파이썬 `SnakeEngine` + `ctx` 어댑터. 몸통을 `deque`로 관리(`appendleft` 머리/`pop` 꼬리), 먹으면 꼬리 유지로 1칸 성장. `_OPPOSITE`로 역방향 금지, 충돌 시 이번 턴 빠질 꼬리를 제외해 정확 판정. 고정 틱(`SNAKE_TICK_MS`).

## 4. 하드웨어 추상화 계층 (hardware/)

> 공통 패턴: 라이브러리 임포트를 `try/except`로 감싸 `_*_AVAILABLE` 플래그를 세우고, **실제 클래스 + 더미(no-op) 클래스**를 함께 정의. 외부는 `create_*()` 팩토리만 호출하며 ①`USE_DUMMY_HARDWARE` ②라이브러리 부재 ③생성 예외 중 하나면 더미 반환.

| 모듈 | 부품/기술 | 핵심 구현 |
|---|---|---|
| `display.py` | OLED SSD1306 (luma.oled, I2C 0x3C) | pygame 프레임 → PIL → `show(image)`로 128×64 전송 |
| `joystick.py` | 조이스틱 + MCP3008 (SPI) | ADC 0.0~1.0 → -1~1 변환, 데드존 방향 판정. SW는 GPIO25 디지털+내부풀업 |
| `ultrasonic.py` | HC-SR04 | `gpiozero.DistanceSensor`, **`partial=True`** 로 미연결 시 무한대기 방지, 실패 시 먼 거리로 폴백 |
| `buttons.py` | 푸시버튼 ×3 | `Button(pull_up=True)`, `pressed_edges()`가 '새로 눌린' 인덱스만(엣지 트리거) |
| `buzzer.py` | 피에조 부저 | `TonalBuzzer(octaves=2)`, 범위 클램프 후 재생, 예외 삼킴 |
| `leds.py` | LED ×4 | `gpiozero.LED` 묶음, `set(index,on)`·`all_off()` |
| `keypad.py` | 4×4 매트릭스 키패드 | **매트릭스 스캔**: 한 행만 LOW로 내리고 각 열(풀업) 읽기, `pressed_edges()` 차집합 |
| `segment_display.py` | 4자리 7세그 + 74HC595 | **lgpio 비트뱅** + 백그라운드 스레드 **자릿수 멀티플렉싱**(한 자리씩 빠르게 순회), 글리프 인코딩 |
| `environment.py` | LDR + BMP180 | §4.1 참고 |

### 4.1 환경 센서 (`environment.py`) — LDR(조도) + BMP180(온도·기압)
- **기능**: 두 환경 입력을 게임에 제공 — 조도로 **수면/기상(낮·밤)**, 온도로 **더위/추위** 표정.
- **구현**:
  - **조도(LDR)**: 조이스틱과 같은 **MCP3008 CH2**(`config.ADC_CHANNEL_LIGHT`)에서 0.0(캄캄)~1.0(밝음)을 읽음. `LIGHT_INVERT`로 배선에 따른 반전 보정(`environment.py:180-185`).
  - **온도·기압(BMP180)**: 외부 라이브러리 없이 `smbus`로 직접 드라이버 구현(`_BMP180`, `environment.py:67-147`) — 데이터시트의 보정 계수(AC1~MD)와 온도/기압 공식을 그대로 구현. I2C는 OLED와 공유(0x77).
  - `read()`가 한 시점 스냅샷(`EnvReading`)을 반환하고, `mood_signals()`로 무드 룰 엔진에 온도·수면·조도 신호를 넘긴다. **습도는 BMP180 미지원** → 항상 None(BME280 교체 시 그 자리에 채움).
  - 센서 미연결 시 `DummyEnvironment`(모든 값 None=중립)로 폴백.

## 6. 저장 · 상점 (storage/ · shop/)

### `storage/save.py` — 세이브 (JSON / SQLite)
- **구현**: 공통 포맷 `{"saved_at":epoch, "character":{...}}`을 `config.SAVE_BACKEND`에 따라 JSON(`write_text`/`read_text`) 또는 SQLite(단일 행 upsert)로 영속화. `load_character()`는 세이브가 없으면 새 캐릭터, 있으면 `Character.from_dict`로 복원해 (캐릭터, 저장시각) 반환. 손상/없음은 예외를 잡아 안전 폴백.

- **구현**: `config.SHOP_ITEMS`에서 불변 `Item`(id/name/price/category/effect) 목록 로드. `buy()`가 재화 차감 후 `apply_effects()`로 능력치 적용. `consumable`은 즉시 소모, 코스튬·기능 아이템은 인벤토리에 추가.
