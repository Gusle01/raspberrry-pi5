# 🗺️ RasPet 개발 로드맵

계획서를 바탕으로 한 단계별 개발 마일스톤입니다. 각 단계의 체크리스트 항목은 그대로
**GitHub Issues**로 등록하여 작업을 관리하는 것을 권장합니다. (활용법은 맨 아래 참고)

---

## 1단계 · 요구사항 분석 및 사양 확정
> 무엇을 만들지 확정하는 단계. 이후 모든 결정의 기준이 된다.

- [ ] 디스플레이 사양 확정 (흑백 SSD1306 / 컬러 SSD1351·ST7789 중 선택)
- [ ] 화면 해상도 결정 및 픽셀아트 에셋 규격 정의
- [ ] ADC 모듈(MCP3008) 및 microSD 카드 등 추가 부품 확보
- [ ] GPIO 핀맵 설계 (초음파·조이스틱·OLED·부저 핀 배정)
- [ ] 능력치/재화 밸런스 초안 작성 (스탯 범위, 감소 속도, 보상량)

## 2단계 · 하드웨어 연동 검증
> 각 장치가 라즈베리파이에서 개별적으로 동작하는지 먼저 확인한다.

- [ ] Raspberry Pi OS 64-bit 설치 및 개발 환경 구축
- [ ] 카메라(picamera2) 영상 캡처 테스트
- [ ] OLED 출력 테스트 (luma.oled로 도형·텍스트 표시)
- [ ] 초음파 센서(HC-SR04) 거리 측정 테스트
- [ ] 조이스틱 + MCP3008 ADC 값 읽기 테스트
- [ ] (선택) 부저 효과음 출력 테스트

## 3단계 · 게임 코어
> 미니게임을 얹을 수 있는 게임 골격을 만든다.

- [ ] Pygame 기반 메인 게임 루프 구현
- [ ] 씬(상태) 머신 구현 (메뉴 → 육성 → 미니게임 → 상점 → 엔딩)
- [ ] 캐릭터 데이터 모델 및 스탯 시스템
- [ ] 시간 경과 요소(포만도·청결도 감소) 구현
- [ ] 저장/불러오기 (SQLite 또는 JSON) — 마지막 플레이 시각 저장 포함
- [ ] OLED 렌더링 파이프라인 (Pygame 프레임 → 디스플레이 전송)

## 4단계 · 미니게임 구현

- [ ] 초음파 점프 게임 *(제안)*
- [ ] 스네이크 / 떨어지는 아이템 받기 *(제안)*

## 📋 GitHub로 작업 관리하기

이 로드맵을 깃허브에서 효율적으로 관리하는 방법입니다.

**1. 마일스톤 등록** — 저장소 → `Issues` → `Milestones` → `New milestone`
위 1~6단계를 각각 마일스톤으로 만들어 둡니다.

**2. 이슈 등록** — 각 체크리스트 항목을 `Issues` → `New issue`로 등록하고,
해당 마일스톤과 라벨(예: `hardware`, `vision`, `ai`, `game`, `enhancement`)을 붙입니다.

**3. 프로젝트 보드(선택)** — `Projects`에서 칸반 보드(To do / In progress / Done)를 만들어
이슈를 끌어다 놓으면 진행 상황을 한눈에 볼 수 있습니다.

**4. 커밋과 이슈 연결** — 커밋 메시지나 PR에 `#이슈번호`를 적으면 자동으로 연결되고,
`Closes #12`처럼 적으면 머지 시 해당 이슈가 자동으로 닫힙니다.
