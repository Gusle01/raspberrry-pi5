## 1. 프로젝트 개요

### 1.1 추진 배경 및 목적

본 프로젝트는 단일 보드 컴퓨터인 라즈베리파이 5를 기반으로, 사용자가 자신만의 캐릭터를 키우는 휴대형 육성 게임기를 직접 제작하는 것을 목표로 한다. 1990년대 휴대용 전자 완구인 '다마고치'의 캐릭터 돌봄 요소와, 캐릭터의 능력치를 성장시키는 육성 시뮬레이션 게임 '프린세스 메이커'의 성장·엔딩 구조를 결합하여, 하나의 임베디드 기기 안에서 완결적으로 동작하는 게임을 구현한다.

이 프로젝트는 단순한 소프트웨어 개발을 넘어, 카메라·초음파 센서·조이스틱 등 다양한 입력 장치를 OLED 출력 장치와 통합하고, 컴퓨터 비전과 게임 인공지능을 하나의 시스템으로 묶어내는 데 의의가 있다. 이를 통해 하드웨어 제어, 영상 처리, 게임 로직 설계, 데이터 영속화 등 임베디드 시스템 개발의 전 과정을 경험하는 것을 목적으로 한다.

### 1.2 프로젝트 요약

사용자는 게임기를 켜면 OLED 화면에 등장하는 자신의 캐릭터를 마주한다. 캐릭터는 시간이 지남에 따라 배가 고파지고 스트레스를 받으며, 사용자는 미니게임을 플레이해 게임 내 재화를 획득하고, 그 재화로 캐릭터에게 밥을 주거나 놀아 주고, 상점에서 코스튬·아이템을 구매하여 캐릭터를 돌보고 꾸민다. 캐릭터는 다양한 활동과 미니게임을 통해 체력·지력·매력 등의 능력치가 성장하며, 최종 능력치 조합에 따라 서로 다른 엔딩에 도달한다.

핵심 미니게임으로는 카메라로 사용자의 손동작을 인식하는 '가위바위보', 인공지능과 대결하는 '오목', 그리고 초음파 센서·조이스틱을 활용한 반응형 미니게임을 포함한다.

## 2. 프로젝트 목표 및 학습 목표

- **임베디드 통합:** 라즈베리파이 5에 다종(多種)의 입출력 장치(카메라, 초음파 센서, 조이스틱, OLED)를 연결하고 하나의 애플리케이션으로 통합 제어한다.
- **컴퓨터 비전 응용:** 카메라 영상에서 사용자의 손 모양을 실시간으로 인식·분류하여 게임 입력으로 활용한다.
- **게임 인공지능 구현:** 탐색 기반 알고리즘으로 오목 대전 상대 AI를 구현하고, 난이도를 조절할 수 있도록 설계한다.
- **게임 엔진·GUI 설계:** 제한된 해상도의 디스플레이에서 동작하는 게임 루프, 메뉴, 캐릭터 렌더링 시스템을 구현한다.
- **데이터 영속화:** 캐릭터의 능력치·소지 재화·아이템 등 게임 상태를 저장하여 전원을 꺼도 진행 상황이 유지되도록 한다.
- **HW–SW 협업 설계:** 하드웨어 제약(전원, 입력 방식, 해상도)을 고려하여 소프트웨어 구조를 설계하는 경험을 쌓는다.

## 3. 시스템 구성

전체 시스템은 입출력 장치로 이루어진 하드웨어 계층, 라즈베리파이 위에서 동작하는 소프트웨어 계층, 그리고 실제 게임 기능을 담당하는 애플리케이션 계층의 3계층으로 구성된다.

### 3.1 하드웨어 구성

메인 보드인 라즈베리파이 5(RAM 4GB)를 중심으로, 사용자 입력을 받는 카메라 모듈·초음파 센서·조이스틱과, 게임 화면을 출력하는 OLED 디스플레이가 연결된다. 카메라 모듈은 라즈베리파이의 CSI 포트에 연결하며, 초음파 센서·조이스틱·OLED는 GPIO 핀에 연결한다. 다만 일반적인 아날로그 조이스틱은 전압값을 출력하므로, 아날로그 입력 단자가 없는 라즈베리파이에서 값을 읽기 위해서는 별도의 ADC(아날로그-디지털 변환) 모듈이 필요하다(자세한 내용은 6장 부품 목록 참조).

### 3.2 소프트웨어 구성 및 기술 스택

운영체제는 Raspberry Pi OS(64-bit)를 사용하며, 모든 기능은 Python 3로 구현한다.

| 구분 | 사용 기술 / 라이브러리 | 용도 |
|---|---|---|
| 운영체제 | Raspberry Pi OS (64-bit) | 전체 시스템 구동 환경 |
| 개발 언어 | Python 3 | 게임 로직·하드웨어 제어 통합 구현 |
| 게임 엔진 | Pygame | 메인 게임 루프, 입력 처리, 화면(프레임) 렌더링 |
| 디스플레이 | luma.oled / Pillow | 렌더링한 프레임을 OLED로 전송·출력 |
| 컴퓨터 비전 | OpenCV, MediaPipe, picamera2 | 카메라 영상 캡처 및 가위바위보 손 인식 |
| 게임 AI | 자체 구현 (Minimax + α-β 가지치기) | 오목 대전 상대 인공지능 |
| 하드웨어 제어 | gpiozero / lgpio | 초음파 센서·조이스틱·버저 등 GPIO 제어 |
| 데이터 저장 | SQLite 또는 JSON | 캐릭터 상태·재화·아이템 영속 저장 |

## 4. 주요 기능

게임은 크게 ① 캐릭터 육성 시스템, ② 미니게임, ③ 상점·재화 시스템의 세 축으로 구성되며, 여기에 ④ 추가 제안 기능을 더한다.

### 4.1 캐릭터 육성 시스템

프린세스 메이커의 능력치 성장 구조를 기반으로 한다. 캐릭터는 여러 능력치(스탯)를 가지며, 미니게임과 활동을 통해 이를 성장시킨다. 동시에 다마고치의 '돌봄' 요소를 결합하여, 시간이 지나면 포만도·청결도·행복도가 서서히 감소하고, 이를 방치하면 능력치 성장 효율이 떨어지거나 부정적 이벤트가 발생한다.

#### 4.1.1 능력치(스탯) 구성

| 능력치 | 설명 | 주요 상승 경로 |
|---|---|---|
| 체력 | 캐릭터의 기초 건강. 낮으면 병에 걸리기 쉽다. | 휴식, 운동 미니게임, 식사 |
| 근력 | 신체 능력. 일부 미니게임 성과에 영향. | 운동·반응형 미니게임 |
| 지력 | 지적 능력. 오목 등 두뇌 게임 보상에 영향. | 오목, 학습 활동 |
| 매력 | 외형·호감도. 일부 엔딩 분기 조건. | 코스튬 착용, 특정 이벤트 |
| 감수성 | 예술·감성 능력. 엔딩 다양성에 기여. | 놀기, 특정 이벤트 |
| 스트레스 | 높을수록 능력치 성장 저하·부정 이벤트 유발. | (감소) 놀기, 휴식 |
| 행복도 | 캐릭터의 만족도. 돌봄 상태를 종합 반영. | 식사·놀기·청결 유지 |

#### 4.1.2 시간 경과 요소 (다마고치 결합)

포만도(배고픔)와 청결도는 실제 시간이 지남에 따라 점진적으로 감소한다. 포만도가 낮으면 식사를, 청결도가 낮으면 씻기/돌보기를 통해 회복하며, 이를 오래 방치하면 행복도와 체력이 떨어진다. 이를 위해 게임은 마지막 플레이 시각을 저장해 두고, 다시 실행했을 때 경과 시간만큼 상태를 갱신한다.

#### 4.1.3 육성 루프

기본 게임 흐름은 '활동 선택 → 능력치·상태 변화 → 캐릭터 돌보기 → (반복)'의 순환 구조를 가진다. 미니게임 플레이는 이 활동의 핵심 수단이며, 게임에서 이기면 재화와 능력치 상승을 동시에 얻는다.

### 4.2 미니게임

미니게임은 재화 획득의 주요 수단이며, 각 게임은 서로 다른 입력 장치와 기술을 활용하도록 설계한다. 1~2번 게임은 필수 게임이고, 3~5번 게임은 보유 부품(초음파 센서·조이스틱·카메라)을 활용한 추가 제안이다.

- **4.2.1 가위바위보 (카메라 활용 · 필수):** 카메라 모듈로 사용자의 손을 촬영하여 가위·바위·보 중 무엇을 냈는지 인식하고, 컴퓨터(게임)의 무작위 선택과 비교해 승패를 판정한다. 사용자가 이기면 재화를 지급한다. (기술 세부: 5.1절)
- **4.2.2 오목 (AI 대전 · 필수):** 바둑판 위에서 인공지능과 번갈아 돌을 놓아 먼저 다섯 개를 일렬로 연결하면 이기는 게임이다. 탐색 기반 알고리즘으로 구현하며(5.2절), 탐색 깊이를 조절하여 난이도를 단계별로 제공한다. 조이스틱으로 커서를 이동해 돌을 놓는다.
- **4.2.3 초음파 점프 게임 (제안):** 초음파 센서 위에서 손의 높이(거리)를 조절하여 캐릭터의 점프 높이를 제어하고, 다가오는 장애물을 피하는 반응형 게임이다. '플래피 버드'와 유사한 조작을 손동작으로 구현한다.
- **4.2.4 조이스틱 미니게임 (제안):** 클래식 '스네이크', 그리고 '떨어지는 아이템 받기'. 두 게임 모두 단순한 흑백/저해상도 화면에 잘 어울려 OLED 환경에 적합하다.
- **4.2.5 카메라 색깔 찾기 (제안):** 화면이 특정 색을 제시하면, 사용자가 그 색을 띤 주변 사물을 카메라에 비추어 제한 시간 안에 인식시키는 게임이다. OpenCV의 색상 공간 변환과 임계값 처리로 구현한다.

### 4.3 상점 및 재화 시스템

미니게임으로 획득한 재화는 캐릭터를 돌보거나 상점에서 아이템을 구매하는 데 사용한다. 재화의 순환 구조는 '미니게임 승리 → 재화 획득 → 식사·놀기 소비 또는 상점 구매 → 캐릭터 성장·꾸미기'로 이어진다.

| 분류 | 예시 품목 | 효과 |
|---|---|---|
| 코스튬 | 모자, 옷, 배경 테마 등 | 캐릭터 외형 변경, 매력 능력치 보정 |
| 소비 아이템 | 먹이, 간식, 장난감 | 포만도·행복도 회복, 스트레스 감소 |
| 기능 아이템 | 능력치 초기화권 | 특정 능력치를 초기 상태로 재설정 |
| 기능 아이템 | 성장 부스터 | 일정 시간 능력치 상승량 증가 |

### 4.4 추가 제안 기능 (프린세스 메이커 응용)

- **다중 엔딩 시스템:** 최종 능력치 조합에 따라 학자·예술가·모험가 등 서로 다른 결말에 도달하게 하여, 반복 플레이 동기를 부여한다.
- **랜덤 이벤트:** 플레이 중 확률적으로 발생하는 사건(예: 길에서 재화를 줍거나, 감기에 걸려 체력이 감소)으로 변화를 준다.
- **도전 과제(업적):** '가위바위보 10연승', '능력치 합계 달성' 등 목표를 제시하고 달성 시 보상을 지급한다.
- **성장 단계(진화):** 캐릭터가 성장함에 따라 외형이 단계별로 변화하여, 키우는 재미와 성취감을 더한다.
- **시간대 반영:** 라즈베리파이의 시스템 시각을 활용해 아침·밤에 따라 화면 분위기나 발생 이벤트를 다르게 한다.
- **감정 표현:** 포만도·스트레스 등 상태에 따라 캐릭터의 표정·애니메이션이 변하여 돌봄의 피드백을 시각적으로 전달한다.

## 5. 핵심 기술 상세

### 5.1 가위바위보 손 인식

카메라 영상에서 손을 인식하는 방법으로는 구글의 MediaPipe Hands 라이브러리 사용을 우선 검토한다. MediaPipe는 손에서 21개의 관절(랜드마크) 좌표를 실시간으로 추출해 주며, 별도의 데이터 수집이나 모델 학습 없이도 손가락이 펴졌는지 접혔는지를 판별할 수 있다.

- **바위:** 모든 손가락이 접힌 주먹 형태 (펴진 손가락 0개)
- **보:** 모든 손가락이 펴진 형태 (펴진 손가락 5개)
- **가위:** 검지와 중지만 펴진 형태 (펴진 손가락 2개)

만약 라즈베리파이 환경에서 MediaPipe 설치·구동에 제약이 있을 경우를 대비하여, OpenCV의 윤곽선(contour)과 볼록 결함(convexity defects) 분석으로 손가락 개수를 세는 방식을 대체 방안으로 준비한다.

### 5.2 오목 대전 AI

오목과 같은 완전정보 보드게임에서는 깊은 학습 모델을 따로 만들지 않고도, 고전적인 탐색 알고리즘인 Minimax에 알파-베타 가지치기를 결합하여 충분히 강한 상대를 구현할 수 있다. 이 방식은 학습 데이터나 모델 파일이 필요 없고 동작이 결정적이어서, 자원이 제한된 임베디드 환경에 적합하다. 각 수의 가치는 돌의 연결 형태(열린 2·열린 3·4 등)에 점수를 부여하는 평가 함수로 계산하며, 탐색 깊이를 조절해 '쉬움/보통/어려움' 난이도를 제공한다.

> **참고:** 초기 아이디어에서 언급한 '인공지능 모델'은 신경망 학습 모델을 의미할 수 있으나, 본 프로젝트 규모에서는 위의 탐색 기반 방식이 구현 난이도·안정성·성능 면에서 더 현실적이다. 학습형 AI(예: 강화학습)는 향후 확장 과제로 남겨 둔다.

### 5.3 디스플레이 선정 검토

'캐릭터를 꾸미는' 콘셉트에서 디스플레이의 해상도와 색상은 표현력에 직접 영향을 준다.

- **흑백 OLED (예: SSD1306/SSD1309, 128×64):** 표현은 제한적이나 다마고치 특유의 레트로 픽셀아트 감성과 잘 어울린다. 화면이 작을 경우 2.42인치급의 비교적 큰 흑백 OLED를 권장한다.
- **컬러 디스플레이 (예: SSD1351 컬러 OLED 128×128, 또는 ST7789 계열 소형 컬러 TFT):** 코스튬·캐릭터 표현이 풍부하여 꾸미기 콘셉트에 유리하다.

어느 방향이든 Pygame으로 작은 해상도의 프레임을 그린 뒤 디스플레이 드라이버로 전송하는 구조는 동일하게 적용할 수 있다. 최종 사양은 1단계(요구사항 분석)에서 확정한다.

## 7. 기대 효과

첫째, 카메라·센서·디스플레이를 아우르는 임베디드 시스템을 처음부터 끝까지 직접 설계·구현함으로써 하드웨어와 소프트웨어를 통합하는 실무 역량을 기른다. 둘째, 컴퓨터 비전(손 인식)과 게임 인공지능(오목)이라는 두 가지 핵심 응용 기술을 하나의 작품 안에서 경험한다. 셋째, 제한된 자원(작은 화면, 한정된 입력 장치) 위에서 사용자 경험을 설계하는 과정을 통해, 제약 조건을 고려한 문제 해결 능력을 함양한다. 완성된 결과물은 동작하는 휴대형 게임기로서 그 자체로 전시·시연이 가능한 결과물이 된다.
